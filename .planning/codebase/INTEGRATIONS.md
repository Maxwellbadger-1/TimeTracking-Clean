# External Integrations

## Database

### SQLite (Primary Data Store)
- **Type**: Embedded relational database
- **Library**: `better-sqlite3` (synchronous SQLite driver)
- **Location**:
  - Development: `server/database.db`
  - Production: Configurable via `DATABASE_PATH` env var
- **Configuration**:
  - **WAL Mode**: Enabled for multi-user concurrent access
  - **Foreign Keys**: Enforced at runtime (verified on startup)
  - **Pragmas**: `journal_mode=WAL`, `foreign_keys=ON`
- **Backup**: Automated daily backups via cron (2 AM UTC)
- **Migration System**: Custom SQL migration runner in `server/src/database/migrations/`

**Connection File**: `server/src/database/connection.ts`

**Key Features**:
- Single database file (no external database server needed)
- Local-first architecture
- No network latency
- Automatic backups with retention policy

## Authentication & Authorization

### Session-Based Authentication
- **Library**: `express-session` + `express-session` cookies
- **Storage**: In-memory session store (development), file-based (production)
- **Security**:
  - **bcrypt** for password hashing (cost factor 10)
  - **SESSION_SECRET** environment variable for cookie encryption
  - **HttpOnly cookies** to prevent XSS attacks
  - **SameSite=Lax** cookie attribute
  - **Secure cookies** in production (HTTPS only)

### JWT Tokens (Dual Auth System)
- **Library**: `jsonwebtoken`
- **Use Case**: Desktop app authentication (alongside sessions)
- **Token Lifetime**: Configurable (default: 24h)
- **Storage**: localStorage (desktop app), httpOnly cookies (web)

**Auth Routes**: `server/src/routes/auth.ts`
**Auth Service**: `server/src/services/authService.ts`
**Client Auth Store**: `desktop/src/store/authStore.ts` (Zustand)

## HTTP & API Communication

### Tauri HTTP Client (Desktop App)
- **Plugin**: `@tauri-apps/plugin-http`
- **Purpose**: Cross-origin HTTP requests from desktop app to server
- **Critical Issue Solved**: Browser `fetch()` doesn't send cookies in Tauri cross-origin requests
- **Implementation**: `desktop/src/lib/tauriHttpClient.ts` (`universalFetch` wrapper)

**Usage Pattern**:
```typescript
import { universalFetch } from '../lib/tauriHttpClient';
await universalFetch('http://localhost:3000/api/...', { credentials: 'include' });
```

### CORS Configuration
- **Library**: `cors` middleware
- **Allowed Origins**:
  - Development: `tauri://localhost`, `http://localhost:1420`, `http://localhost:5173`
  - Production: `tauri://localhost` + custom origins via `ALLOWED_ORIGINS` env var
- **Credentials**: `credentials: true` (required for cookies)
- **Methods**: GET, POST, PUT, PATCH, DELETE, OPTIONS

**CORS Config**: `server/src/server.ts:80-100`

### REST API
- **Framework**: Express.js
- **Architecture**: RESTful endpoints
- **Response Format**: JSON
- **Error Handling**: Centralized error middleware
- **Rate Limiting**:
  - API endpoints: 100 requests/15 min
  - Login endpoint: 5 requests/15 min

## Real-Time Communication

### WebSocket Server
- **Library**: `ws` (WebSocket server)
- **Port**: Same as HTTP server (upgrade mechanism)
- **Purpose**: Real-time notifications, live updates
- **Events**:
  - `notification` - Push notifications to clients
  - `overtime_updated` - Broadcast overtime balance changes
  - `time_entry_updated` - Live time entry updates
  - `absence_status_changed` - Absence request approvals/rejections

**WebSocket Implementation**: `server/src/websocket/server.ts`
**Client WebSocket Hook**: `desktop/src/hooks/useWebSocket.ts`

**Authentication**: Session-based (validated on connection upgrade)

## External Services

### Holiday API (Feiertagsapi)
- **Purpose**: German public holidays (federal and state-specific)
- **Auto-Update**: Scheduled yearly via cron
- **Storage**: Local SQLite table (`holidays`)
- **Integration**: `server/src/services/holidayService.ts`

**Functions**:
- `initializeHolidays()` - Load holidays on server start
- `autoUpdateHolidays()` - Scheduled cron job (January 1st)
- Holidays are cached in database, no runtime API calls

**Holiday Table Schema**:
```sql
CREATE TABLE holidays (
  id INTEGER PRIMARY KEY,
  name TEXT NOT NULL,
  date TEXT NOT NULL,
  state TEXT, -- DE-BY, DE-NW, etc.
  createdAt TEXT DEFAULT (datetime('now'))
);
```

### GitHub Releases (Auto-Update)
- **Plugin**: `@tauri-apps/plugin-updater`
- **Update Endpoint**: `https://github.com/Maxwellbadger-1/TimeTracking-Clean/releases/latest/download/latest.json`
- **Signature Verification**: Public key embedded in `tauri.conf.json`
- **Update Check**: On app startup + manual check in settings
- **Platforms**: macOS (DMG), Windows (EXE, MSI), Linux (AppImage, DEB)

**Update Manifest**: `latest.json` (auto-generated by GitHub Actions on release)

**Update Flow**:
1. App checks GitHub Releases for `latest.json`
2. Compares version in manifest with current app version
3. Downloads signed binary if newer version available
4. Verifies signature with embedded public key
5. Prompts user to install update

**Implementation**: `desktop/src/components/settings/UpdateChecker.tsx`

## Security Hardening

### HTTP Security Headers (Helmet)
- **Library**: `helmet`
- **Headers**:
  - Content-Security-Policy (XSS protection)
  - X-Frame-Options (clickjacking protection)
  - X-Content-Type-Options (MIME sniffing protection)
  - Strict-Transport-Security (HTTPS enforcement)

### Rate Limiting
- **Library**: `express-rate-limit`
- **API Limit**: 100 requests per 15 minutes
- **Login Limit**: 5 requests per 15 minutes
- **Protection**: Brute-force attack prevention

**Middleware**: `server/src/middleware/rateLimits.ts`

## Environment-Specific Configuration

### Development
- CORS: All localhost origins allowed
- Logging: Pretty-print to console
- Database: Local SQLite file
- Session: In-memory store
- Future dates: Allowed for time entries

### Production
- CORS: Strict origin whitelist
- Logging: JSON to PM2 logs
- Database: Production SQLite file
- Session: File-based store
- Future dates: Blocked for time entries
- SESSION_SECRET: Required environment variable
- TZ: Must be `Europe/Berlin` (overtime calculations)

### Staging (Green Server)
- CORS: Same as production
- Database: Separate `database.green.db`
- Port: 3001 (vs. 3000 for production)
- ENV vars: Prefixed with `GREEN_` for PM2 ecosystem config
