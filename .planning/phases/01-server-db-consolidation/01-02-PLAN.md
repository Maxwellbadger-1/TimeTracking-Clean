Run the lsof investigation via SSH. The command handles two possible PID detection methods (pgrep and pm2 pid):

```bash
ssh -i "/c/Users/maxfe/Maxflow Software/Projekte/Stiftung TimeTracker/TimeTracking-Clean/.ssh/oracle_server.key" \
  -o StrictHostKeyChecking=no \
  ubuntu@129.159.8.19 \

   pm2 list && \
   echo '' && \
   echo '=== Blue Server PID ===' && \
   PM2_PID=\$(pgrep -f 'timetracking-server' | head -1) && \
   echo \"PID: \$PM2_PID\" && \
   echo '' && \
   echo '=== Open DB files (lsof) ===' && \
   lsof -p \$PM2_PID 2>/dev/null | grep '\.db' && \
   echo '' && \
   echo '=== All DB files on server ===' && \
   ls -lh /home/ubuntu/*.db 2>/dev/null && \
   ls -lh /home/ubuntu/TimeTracking-Clean/server/*.db 2>/dev/null && \
   echo '' && \
   echo '=== Disk Space ===' && \
   df -h /home/ubuntu"
```
