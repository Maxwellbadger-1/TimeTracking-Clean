New step to insert (exact YAML):

```yaml
      - name: Verify DB path post-deploy
        uses: appleboy/ssh-action@v1.0.3
        with:
          host: ${{ secrets.ORACLE_HOST }}
          username: ${{ secrets.ORACLE_USER }}
          key: ${{ secrets.ORACLE_SSH_KEY }}
          port: 22
          script: |
            set -e
            echo "Verifying DB path..."
            PM2_PID=$(pm2 pid timetracking-server)
            if [ -z "$PM2_PID" ] || [ "$PM2_PID" = "0" ]; then
              echo "ERROR: PM2 process 'timetracking-server' not found or not running"
              exit 1
            fi
            echo "PM2 PID: $PM2_PID"
            DB_PATH=$(lsof -p "$PM2_PID" | grep '\.db$' | awk '{print $NF}' | head -1)
            EXPECTED="/home/ubuntu/databases/production.db"
            if [ "$DB_PATH" = "$EXPECTED" ]; then
              echo "DB verification PASSED: $DB_PATH"
            else
              echo "DB verification FAILED: expected $EXPECTED, got: $DB_PATH"
              exit 1
            fi
```

Key details:
- Uses `pm2 pid timetracking-server` (NOT pgrep -f) per Phase 1 decision
- Uses `grep '\.db$'` with dollar anchor to exclude WAL/SHM files (per RESEARCH Pitfall 1)
- Checks for empty PID or "0" before calling lsof (per RESEARCH Pitfall 2)
- Reuses same SSH secrets as existing deploy step (ORACLE_HOST, ORACLE_USER, ORACLE_SSH_KEY)
- Step placement: after the deploy SSH step, before "Notify on success"
