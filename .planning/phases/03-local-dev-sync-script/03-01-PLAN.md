Create `scripts/sync-dev-db.sh` — a bash script that downloads the production SQLite DB for local development. The script must be Windows/Git Bash compatible (no xattr, no macOS-only commands, no sqlite3 CLI).

**Script structure (10 steps):**

1. **Shebang and set -e:**
   ```
   #!/bin/bash
   set -e
   ```

2. **Color definitions** (same as sync-prod.sh): RED, GREEN, YELLOW, BLUE, NC variables.

3. **Path resolution using BASH_SOURCE[0]:**
   ```bash
   SCRIPT_DIR="$(cd "$(dirname "${BASH_SOURCE[0]}")" && pwd)"
   PROJECT_ROOT="$(cd "$SCRIPT_DIR/.." && pwd)"
   ```

4. **Configuration variables:**
   - `SSH_KEY="$PROJECT_ROOT/.ssh/oracle_server.key"` (NOT `../.ssh/` like old script)
   - `PROD_HOST="ubuntu@129.159.8.19"`
   - `PROD_DB_PATH="/home/ubuntu/databases/production.db"` (Phase 2 canonical path)
   - `LOCAL_DB="$PROJECT_ROOT/server/database.db"` (per ROADMAP SC-2)
   - `TIMESTAMP=$(date +%Y%m%d_%H%M%S)`
   - `TEMP_DB="/tmp/prod_sync_${TIMESTAMP}.db"`

5. **Trap for cleanup:**
   ```bash
   trap 'rm -f "$TEMP_DB"' EXIT
   ```

6. **Pre-flight checks (3 checks):**
   - Check SSH key exists: `[ -f "$SSH_KEY" ]` or exit 1 with message
   - `chmod 600 "$SSH_KEY" 2>/dev/null || true`
   - Check better-sqlite3 installed: `[ -d "$PROJECT_ROOT/server/node_modules/better-sqlite3" ]` or exit 1 with message "Run: cd server && npm install"
   - Test SSH connectivity: `ssh -i "$SSH_KEY" -o ConnectTimeout=10 -o StrictHostKeyChecking=no "$PROD_HOST" "echo ok" > /dev/null 2>&1` or exit 1

7. **Backup existing local DB (if exists):**
   ```bash
   if [ -f "$LOCAL_DB" ]; then
     BACKUP="$PROJECT_ROOT/server/database.db.backup.${TIMESTAMP}"
     cp "$LOCAL_DB" "$BACKUP"
     echo "Backed up to: $BACKUP"
   fi
   ```

8. **SCP download to temp path:**
   ```bash
   scp -i "$SSH_KEY" -o StrictHostKeyChecking=no "$PROD_HOST:$PROD_DB_PATH" "$TEMP_DB"
   ```

9. **Integrity check via node + better-sqlite3 (NOT sqlite3 CLI):**
   ```bash
   INTEGRITY=$(node -e "
     try {
       const D = require('$PROJECT_ROOT/server/node_modules/better-sqlite3');
       const db = new D('$TEMP_DB', {readonly: true});
       const r = db.pragma('integrity_check');
       db.close();
       process.stdout.write(r[0].integrity_check);
     } catch(e) {
       process.stderr.write(e.message);
       process.exit(1);
     }
   " 2>&1)
   if [[ "$INTEGRITY" != "ok" ]]; then
     echo "ERROR: integrity_check returned: $INTEGRITY"
     exit 1
   fi
   ```

10. **Move temp to final + summary:**
    ```bash
    mv "$TEMP_DB" "$LOCAL_DB"
    ```
    Then print summary via node -e + better-sqlite3:
    ```bash
    node -e "
      const D = require('$PROJECT_ROOT/server/node_modules/better-sqlite3');
      const db = new D('$LOCAL_DB', {readonly: true});
      const u = db.prepare('SELECT COUNT(*) as cnt FROM users WHERE deletedAt IS NULL').get();
      const e = db.prepare('SELECT MAX(date) as d FROM time_entries').get();
      db.close();
      console.log('Users: ' + u.cnt + ' | Latest time entry: ' + (e.d || 'none'));
    "
    ```
    End with a note: "To use this DB with npm run dev, set DATABASE_PATH=./database.db in server/.env.development"

**Step labels:** Use numbered steps [1/6] through [6/6] with color output matching the existing sync-prod.sh style (SSH check, Backup, Download, Integrity check, Install, Summary).

**Do NOT:** Use `sqlite3` CLI (not available on Windows), use `xattr`, use macOS-only commands, add npm dependencies, write to `server/database/development.db`.

**Do:** Use `node -e` with explicit require path `$PROJECT_ROOT/server/node_modules/better-sqlite3`, quote ALL path variables, use `set -e` and trap for cleanup.
